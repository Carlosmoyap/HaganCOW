CAPÍTOL 01

Introducció a HTML

El llenguatge d'etiquetatge que dóna estructura al contingut web

FONAMENTS

Què és HTML?

HyperText Markup Language

HTML és el llenguatge d'etiquetatge estàndard per crear pàgines web. Defineix l'estructura i el contingut d'una pàgina utilitzant etiquetes (tags) que el navegador interpreta per mostrar la informació.

Important: HTML no és un llenguatge de programació, sinó d'etiquetatge. No té lògica ni algoritmes, només descriu l'estructura.

Breu Història

1991: Tim Berners-Lee crea HTML
1999: HTML 4.01 es converteix en estàndard
2014: HTML5 amb etiquetes semàntiques
Actualitat: HTML5 és l'estàndard universal

Característiques Clau

1

Etiquetes (Tags)

Elements que envolten el contingut per donar-li significat

2

Atributs

Proporcionen informació addicional sobre les etiquetes

3

Estructura Jeràrquica

Els elements s'imbrincan formant un arbre DOM

4

Independència de Plataforma

Funciona a qualsevol navegador i dispositiu

Com funciona?

L'usuari sol·licita una pàgina → El servidor envia l'HTML → El navegador l'interpreta i renderitza → Es mostra el contingut estructurat

ESTRUCTURA

Estructura Bàsica d'un Document HTML

index.html 
<!DOCTYPE html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <meta name="viewport"
          content="width=device-width, initial-scale=1.0">
    <title>Títol de la Pàgina</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <header>
        <h1>Benvinguts</h1>
    </header>
    <main>
        <p>Contingut principal</p>
    </main>
    <footer>
        <p>© 2024</p>
    </footer>
</body>
</html>

<!DOCTYPE html>

Declara que és un document HTML5. Sempre ha de ser la primera línia.

<html>

Element arrel que conté tot el document. L'atribut lang defineix l'idioma.

<head>

Conté metadades: charset, viewport, títol, enllaços a CSS i scripts.

<body>

Conté tot el contingut visible: text, imatges, enllaços, formularis...

Metadades Importants

  • charset: Codificació UTF-8
  • viewport: Adaptació mòbil
  • title: Títol de la pestanya

TAGS ESSENCIALS

Etiquetes HTML Fonamentals

Títols: <h1> - <h6>

Bloc

Defineixen títols i subtítols amb diferents nivells d'importància.

<h1>Títol Principal</h1>
<h2>Subtítol</h2>
<h3>Secció</h3>

Només un <h1> per pàgina per SEO

Paràgrafs: <p>

Bloc

Defineixen blocs de text. Els navegadors afegeixen espai entre paràgrafs.

<p>Aquest és un paràgraf</p>
<p>Un altre paràgraf</p>

Llistes: <ul>, <ol>, <li>

Bloc

<ul> per llistes desordenades, <ol> per ordenades.

<ul>
  <li>Element 1</li>
  <li>Element 2</li>
</ul>

Enllaços: <a>

Inline

Creen enllaços a altres pàgines. L'atribut href és obligatori.

<a href="https://exemple.com">Visita</a>
<a href="#seccio">Anar a secció</a>
<a target="_blank">Nova pestanya</a>

Atributs: href, target, title, download

Imatges: <img>

Inline

Etiqueta buida. src i alt són obligatoris.

<img src="foto.jpg"
     alt="Descripció"
     width="300">

Atributs: src, alt, width/height

CAPÍTOL 02

HTML Semàntic

Donar significat a l'estructura per millorar l'accessibilitat i SEO

HTML5 MODERN

Etiquetes Semàntiques HTML5

<header>

Capçalera d'una pàgina o secció.

<header>
  <h1>Logo</h1>
  <nav>...</nav>
</header>

<nav>

Enllaços de navegació principals.

<nav>
  <a href="/">Inici</a>
</nav>

<main>

Contingut principal. Només un per document.

<main>
  <h1>Títol</h1>
</main>

<section>

Secció temàtica amb títol.

<section>
  <h2>Serveis</h2>
</section>

<article>

Contingut autònom (post, notícia).

<article>
  <h2>Títol</h2>
  <p>Text...</p>
</article>

<aside>

Contingut secundari (sidebar).

<aside>
  <h3>Relacionat</h3>
</aside>

<footer>

Peu de pàgina o secció.

<footer>
  <p>© 2024</p>
</footer>

Beneficis

  • SEO: Millor per motors de cerca
  • Accessibilitat: Lectors de pantalla
  • Mantenible: Codi més llegible

Quan usar <div>?

Per agrupar elements per CSS quan cap etiqueta semàntica té sentit.

CAPÍTOL 04

Introducció a CSS

Estilitzar i donar vida a les pàgines web

FONAMENTS CSS

Què és CSS?

Cascading Style Sheets

CSS és el llenguatge que controla l'aparença visual de les pàgines web. Defineix colors, tipografies, espaçats, layouts i efectes visuals.

Separació de responsabilitats: HTML per l'estructura, CSS per l'aparença

Tres Formes d'Aplicar CSS

1

Inline (evitar)

Dins l'atribut style de l'element

2

Intern

Dins <style> al <head>

3

Extern (recomanat)

Arxiu .css separat amb <link>

styles.css 
/* Selecciona tots els paràgrafs */
p {
  color: #333;
  font-size: 16px;
  line-height: 1.5;
}

/* Selecciona elements amb classe */
.destacat {
  background-color: yellow;
  font-weight: bold;
}

/* Selecciona element amb ID */
#capcalera {
  background: #333;
  color: white;
}

Avantatges CSS Extern

  • Reutilitzable: Un arxiu per múltiples pàgines
  • Mantenible: Canvis en un sol lloc
  • Rendiment: Es cacheja al navegador
  • Separació: HTML net i semàntic

ESTRUCTURA

Sintaxi i Estructura CSS

sintaxi.css 
/*
 * Comentari de múltiples línies
 * Descriu el propòsit del codi
 */

/* Selector d'element */
h1 {
  color: blue;
  font-size: 2em;
}

/* Selector de classe */
.btn {
  background: #007bff;
  color: white;
  padding: 10px 20px;
  border-radius: 4px;
}

/* Selector d'ID */
#navegacio {
  display: flex;
  gap: 20px;
}

/* Selector combinat */
article p {
  text-align: justify;
}

/* Múltiples selectors */
h1, h2, h3 {
  font-family: Arial, sans-serif;
}

Anatomia d'una Regla

selector {
propietat: valor;
}
Selector: Què estilitzar
Propietat: Quin aspecte
Valor: Com aplicar-ho

Regles Importants

  • Cases insensitive (no importa majúscules)
  • Punts i comes obligatoris
  • Clausules ben tancades
  • Comentaris amb /* */

Consells d'Organització

Agrupa per components, utilitza comentaris per seccions, mantén indentació consistent

SELECTORS

Selectors CSS Fonamentals

Selector d'Element

Selecciona tots els elements d'un tipus

/* Tots els paràgrafs */
p {
  color: blue;
}

/* Tots els enllaços */
a {
  text-decoration: none;
}

Selector de Classe

Selecciona elements amb un atribut class específic. Comença amb punt.

/* HTML: <p class="destacat"> */
.destacat {
  background: yellow;
  font-weight: bold;
}

/* Múltiples classes */
.btn.principal {
  background: blue;
}

Selector d'ID

Selecciona un element únic amb atribut id. Comença amb #.

/* HTML: <div id="capcalera"> */
#capcalera {
  background: #333;
  color: white;
}

ID ha de ser únic per pàgina

Selector Universal

Selecciona tots els elements. Útil per resets.

/* Tots els elements */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

Selector de Descendència

Selecciona elements dins d'altres elements (espai)

/* Tots els <p> dins <article> */
article p {
  line-height: 1.8;
}

/* Tots els <li> dins <nav> */
nav li {
  display: inline-block;
}

Selector de Fill Directe

Selecciona només fills directes (>)

/* Només <li> directes de <ul> */
ul > li {
  list-style: none;
}

/* Només <p> directes de <div> */
div > p {
  margin-bottom: 1em;
}

Combinació de Selectors

div.content > p.intro {
  font-style: italic;
}

SELECTORS AVANÇATS

Selectors Avançats i Pseudo-classes

Selectors d'Atribut

Selecciona elements basats en els seus atributs

/* Atribut existeix */
[disabled] { opacity: 0.5; }

/* Valor exacte */
[type="text"] { border: 1px solid #ccc; }

/* Comença amb... */
[href^="https"] { color: green; }

/* Acaba amb... */
[href$=".pdf"] { background: url(icon); }

/* Conté... */
[class*="btn-"] { display: inline-block; }

Pseudo-classes d'Estats

Responen a l'estat o interacció

/* Enllaç no visitat */
a:link { color: blue; }

/* Enllaç visitat */
a:visited { color: purple; }

/* Ratolí a sobre */
a:hover { text-decoration: underline; }

/* Element actiu */
button:active { transform: scale(0.98); }

/* Amb focus */
input:focus { border-color: blue; }

Pseudo-classes Estructurals

Seleccionen elements per posició

/* Primer fill */
li:first-child { font-weight: bold; }

/* Últim fill */
li:last-child { border-bottom: none; }

/* N-èsim fill */
li:nth-child(2n) { background: #f5f5f5; }

/* Des del final */
li:nth-last-child(3) { color: red; }

/* Fill únic */
p:only-child { margin: 0; }

Altres Pseudo-classes

/* Element buit */
div:empty { display: none; }

/* Checkbox marcat */
input:checked { background: blue; }

/* Input deshabilitat */
input:disabled { background: #eee; }

Fórmules nth-child

:nth-child(3) - 3r
:nth-child(2n) - Parells
:nth-child(odd) - Imparells
:nth-child(3n+1) - Cada 3

CONFLICTES

Especificitat i Cascada

La Cascada (Cascading)

CSS determina quins estils s'apliquen quan hi ha conflictes:

1. Origen: Navegador → Usuari → Autor
2. Especificitat: Quina regla és més específica
3. Ordre: L'última regla guanya

Càlcul d'Especificitat

Puntuació: (a, b, c, d) on major és més específic

Inline style 1,0,0,0
ID (#header) 0,1,0,0
Classe/Atribut 0,0,1,0
Element 0,0,0,1
exemple.css 
/* 0,0,0,1 */
p { color: blue; }

/* 0,0,1,0 (guanya) */
.destacat { color: green; }

/* 0,1,0,0 (guanya) */
#titol { color: red; }

/* 0,0,1,1 */
p.destacat { color: orange; }

/* 0,1,1,2 */
article#principal p.destacat {
  color: purple;
}

!important (Evitar)

p { color: blue !important; }

Ús excessiu crea codi difícil de mantenir

Millors Pràctiques

  • Mantenir especificitat baixa
  • Evitar IDs per estilitzar
  • Utilitzar classes reutilitzables
  • No dependre de !important

Layout amb Flexbox i Grid

Sistemes moderns per crear layouts complexos

FLEXBOX

Flexbox: Conceptes Bàsics

Què és Flexbox?

Sistema unidimensional per distribuir elements.

.contenidor { display: flex; }

Direcció i Ajust

.contenidor {
  flex-direction: row; /* column */
  flex-wrap: wrap;
  flex-flow: row wrap;
}

Alineació

.contenidor {
  /* Horitzontal */
  justify-content: center;
  justify-content: space-between;
  /* Vertical */
  align-items: center;
  align-items: flex-start;
}

Espaiat i Fills

.contenidor {
  gap: 20px;
}

.fill {
  flex-grow: 1;
  flex-shrink: 0;
  flex-basis: 200px;
  /* Shorthand */
  flex: 1 0 200px;
}

Centrat Perfecte

.centrat {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
}
Centrat!

Joc Flexbox

flexboxfroggy.com

GRID

CSS Grid: Layouts Bidimensionals

Què és CSS Grid?

Sistema bidimensional amb files i columnes.

.contenidor { display: grid; }

Definir el Grid

.contenidor {
  grid-template-columns: 1fr 2fr 1fr;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: 100px auto 100px;
  gap: 20px;
}

Posicionament

.element {
  grid-column: 1 / 3;
  grid-column: span 2;
  grid-row: 2 / 4;
  grid-area: 1 / 1 / 3 / 3;
}

Àrees de Grid

.contenidor {
  grid-template-areas:
    "header header header"
    "sidebar main main"
    "footer footer footer";
  grid-template-columns: 200px 1fr 1fr;
}

.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }
.footer { grid-area: footer; }

COMPARATIVA

Flexbox vs Grid: Quan Utilitzar Cada Un

Flexbox

Unidimensional: Fila O columna

Ús Recomanat

  • Navegacions
  • Centrat
  • Distribució
  • Components
nav {
  display: flex;
  justify-content: space-between;
}

Punts Forts

  • Alineació
  • Distribució
  • Reordenació

CSS Grid

Bidimensional: Files I columnes

Ús Recomanat

  • Layouts de pàgina
  • Galeries
  • Dashboards
  • Layouts 2D
.layout {
  display: grid;
  grid-template: "header header" auto
    "sidebar main" 1fr
    "footer footer" auto / 200px 1fr;
}

Punts Forts

  • Control 2D
  • Àrees amb nom
  • Responsiu

CAPÍTOL 07

Disseny Responsiu

Adaptar les pàgines a qualsevol dispositiu

RESPONSIVE

Media Queries

Sintaxi Bàsica

Les media queries permeten aplicar estils segons les característiques del dispositiu.

/* Sintaxi bàsica */
@media (condició) {
  /* Estils */
}

/* Exemples */
@media (max-width: 768px) {
  /* Per mòbils */
}

@media (min-width: 769px) and (max-width: 1024px) {
  /* Per tauletes */
}

Breakpoints Comuns

Mòbil petit max-width: 320px
Mòbil max-width: 480px
Tauleta max-width: 768px
Escriptori petit max-width: 1024px
Escriptori min-width: 1025px

Mobile-First vs Desktop-First

Mobile-First (Recomanat)

Comença per mòbil i afegeix estils per a pantalles més grans

/* Estils base (mòbil) */
.contenidor {
  width: 100%;
  padding: 10px;
}

/* Tablet */
@media (min-width: 768px) {
  .contenidor {
    width: 750px;
    margin: 0 auto;
  }
}

Desktop-First

Comença per escriptori i sobreescriu per a pantalles petites

Exemple Pràctic

/* Navegació responsiva */
nav {
  display: flex;
  flex-direction: column;
}

@media (min-width: 768px) {
  nav {
    flex-direction: row;
    justify-content: space-between;
  }
}

Unitats Relatives i Viewport

Percentatge (%)

Relatiu a l'element pare. Útil per widths i layouts fluids.

.contenidor {
  width: 80%;
  margin: 0 auto;
}

.columna {
  width: 50%;
}

em i rem

em: Relatiu a la mida de font del pare

rem: Relatiu a la mida de font de l'arrel

html {
  font-size: 16px;
}

.element {
  font-size: 1.5rem;
  padding: 1em;
  margin: 2rem;
}

Viewport Units

Relatius a la mida de la finestra

vw 1% de l'amplada
vh 1% de l'alçada
vmin 1% del valor més petit
vmax 1% del valor més gran 
.hero {
  height: 100vh;
  width: 100vw;
}

.banner {
  width: 50vw;
}

Unitats Fraccionàries

Per CSS Grid. Fracciona l'espai disponible.

.grid {
  grid-template-columns: 1fr 2fr 1fr;
}

.grid-3 {
  grid-template-columns: repeat(3, 1fr);
}

Quan Utilitzar Cada Un?

rem: Fonts, margins, paddings
em: Components escalables
%: Widths, layouts fluids
vw/vh: Hero sections
fr: Grid layouts

Consell

Estableix font-size: 16px a html i usa rem per tot.

Nomenclatura de Classes

  • Descriptiu i significatiu
  • Kebab-case (classe-nom)
  • Evita noms genèrics
  • BEM: bloc__element--modificador

Consells d'Especificitat

Evitar !important

Ús excessiu crea codi difícil de mantenir.

Mantenir Baixa Especificitat

/* Preferir */
.btn { }

/* Evitar */
div.container > button.btn { }

No Depèn d'IDs

IDs tenen alta especificitat.

Comentaris

 */
